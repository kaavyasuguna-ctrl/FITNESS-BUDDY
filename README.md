💪 FitBuddy AI — IBM Watsonx Powered Fitness & Wellness Web App
A personalised wellness planner built with Python Flask and IBM Watsonx.ai Granite models.
Enter your health profile and receive an AI-generated plan covering fitness habits, nutrition, hydration, exercise routines, and medical safety alerts.

✨ Features
AI Wellness Plan — Generated by ibm/granite-3-3-8b-instruct via Watsonx.ai
⚠️ Critical Medical Alert — Printed first when allergies or conditions are detected
Climate-Aware Hydration — Water targets scaled to your location and lifestyle
Structured Output — Core habits, nutrition, exercise & yoga sections every time
Secure Config — IBM API key stored in .env, never hard-coded
Responsive UI — Clean IBM-design-inspired single-page web app

⚙️ Local Setup
1. Clone / download the project
git clone https://github.com/your-username/fitness-buddy.git
cd fitness-buddy
2. Create a virtual environment
# macOS / Linux
python3 -m venv venv
source venv/bin/activate

# Windows PowerShell
python -m venv venv
.\venv\Scripts\Activate.ps1
3. Install dependencies
pip install -r requirements.txt
4. Configure environment variables
Copy the example file and fill in your credentials:

cp .env.example .env
Edit .env:

IBM_API_KEY=your_ibm_cloud_api_key_here
WATSONX_PROJECT_ID=your_watsonx_project_id_here
WATSONX_URL=https://us-south.ml.cloud.ibm.com
FLASK_SECRET_KEY=any-random-secret-string
FLASK_ENV=development
How to get credentials:

IBM_API_KEY → IBM Cloud Console → Manage → Access (IAM) → API Keys → Create
WATSONX_PROJECT_ID → Watsonx.ai Platform → Your Project → Manage → General → Project ID
WATSONX_URL → Use the region URL matching your Watsonx.ai instance (default: us-south)
5. Run the application
python app.py
Open your browser at http://localhost:5000

🚀 Deployment
Option A — IBM Code Engine (Recommended for IBM Stack)
# Install IBM Cloud CLI + Code Engine plugin
ibmcloud login --apikey $IBM_API_KEY -r us-south
ibmcloud ce project create --name fitbuddy-project
ibmcloud ce project select --name fitbuddy-project

# Deploy from local source
ibmcloud ce app create \
  --name fitbuddy-ai \
  --build-source . \
  --strategy buildpacks \
  --env IBM_API_KEY=$IBM_API_KEY \
  --env WATSONX_PROJECT_ID=$WATSONX_PROJECT_ID \
  --env WATSONX_URL=https://us-south.ml.cloud.ibm.com \
  --env FLASK_SECRET_KEY=your-secret \
  --min-scale 0 \
  --max-scale 3

ibmcloud ce app get --name fitbuddy-ai
Option B — IBM Cloud Foundry
Create manifest.yml:

applications:
  - name: fitbuddy-ai
    memory: 512M
    instances: 1
    buildpack: python_buildpack
    env:
      IBM_API_KEY: your_ibm_cloud_api_key
      WATSONX_PROJECT_ID: your_watsonx_project_id
      WATSONX_URL: https://us-south.ml.cloud.ibm.com
      FLASK_SECRET_KEY: your-secret
ibmcloud cf push
A Procfile is required for Cloud Foundry:

web: gunicorn app:app --bind 0.0.0.0:$PORT
Option C — Docker
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["gunicorn", "app:app", "--bind", "0.0.0.0:5000", "--workers", "2"]
docker build -t fitbuddy-ai .
docker run -p 5000:5000 --env-file .env fitbuddy-ai
Option D — Railway / Render / Heroku
Push code to GitHub
Connect repo to Railway / Render / Heroku
Add all .env variables as environment secrets in the platform dashboard
The gunicorn start command: gunicorn app:app

🔒 Security Notes
.env is listed in .gitignore — never commit it
Use IAM fine-grained API keys scoped to Watsonx.ai only
In production, inject secrets via platform environment variables (not files)
FLASK_ENV should be production in live deployments (disables debug mode)
